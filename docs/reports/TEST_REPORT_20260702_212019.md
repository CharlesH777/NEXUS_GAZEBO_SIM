## 结论

当前版本已经恢复到 **可正常产出探索路径并驱动机器人运动** 的状态，而且相比上一轮，**中后段稳定性明显改善**。

`2026-07-02 22:20` 追加确认了一次执行层修复：之前出现过“`/exporation_path`、`/path`、`/cmd_vel` 都正常，但机器人几乎不动”的现象。根因不是 LRAE，而是 `cmd_vel_to_swerve` 把这台全向/转向底盘错误地退化成了 `steering=0` 的差速式控制，导致轮子在转、车体基本不走。现已改回默认使用 swerve 解算，并把 steering 对正前的保底驱动力降为 `0.0`，避免 Gazebo 里横向顶死。

这次优化分三步:

1. 针对 **更换雷达后 frontier/unknown 分布变化**，放宽 `lrae_planner` 的筛选条件
2. 针对短时 `Replan` 空窗，增加 **exploration path 保活** 和下游 stale/recovery 容忍度
3. 针对 **新雷达视角下通行性地图过于保守**，放宽 `fitplane` 的地形/障碍物判定

修完后，`/exporation_path`、`/look_ahead_goal` 和 `/path` 都能持续发布；短时重规划不再轻易把系统打进 recovery。执行层修复后，`/steering_position_controller/commands` 也恢复为非零，车体真实位移从 `0.0007 m / 9.5 sim s` 恢复到 `0.3231 m / 9.5 sim s`，后续窗口进一步达到 `1.0209 m / 8.9 sim s`。

但要保留一个准确表述:

- **短时与中时窗口内，探索闭环已稳定打通**
- **运行中仍会有 Replan，但现在由 held path 平滑过渡**
- **本轮实测里 recovery 已压到 0 次**
- **还没有做超长 soak test，但已经明显强于上一版**

## 本次修复内容

### 参数与逻辑调整

针对新雷达数据分布，做了以下修改:

- `unknown_num_thre`: `80 -> 16`
- `minrange`: `20.0 -> 4.0`
- 新增 `viewpoint_min_path_distance` 参数，替代原来写死的 `6.0m` 过滤
- 当前启动值: `viewpoint_min_path_distance = 1.0`

这几项直接对应 LRAE 原来的三个过严限制:

1. unknown 区块太小会被直接过滤
2. 离机器人 20m 内的 centroid 会被直接跳过
3. 距离机器人小于 6m 的 viewpoint 会被直接丢弃

对新雷达来说，这三个条件过严，导致 planner 经常“有图但不给路径”。

### 稳定性优化

为减少 `Replan` 期间的抖动，又补了两类优化:

- `lrae_planner` 增加最近一次有效 `/exporation_path` 保活
- 当短时重规划失败时，继续复用最近有效 exploration path，而不是立即断流
- `localPlanner.goalStaleTime`: `1.0 -> 2.0`
- `pathFollower.pathStaleTime`: `1.0 -> 2.0`
- `gen_local_goal.path_stale_timeout_sec`: `1.0 -> 2.0`
- `gen_local_goal.recovery_trigger_cycles`: `10 -> 20`

目标不是让 planner 永远不重规划，而是让它 **重规划时不把下游一并拖进 recovery**。

### 通行性地图放宽

考虑到雷达安装和回波分布已经变化，本轮又把 `fitplane` 的障碍物判定稍微放宽了一档，并改成可通过 ROS 参数显式配置:

- `max_angle_deg`: `40.0 -> 48.0`
- `max_flatness`: `0.01 -> 0.02`
- `safe_obs_dis`: `1.0 -> 0.8`
- `inflation_value`: `99 -> 94`
- `near_obstacle_value`: `120 -> 118`

对应含义:

- 允许更大的局部坡度仍被视为可通行
- 允许更大的平面拟合残差仍被视为可通行
- 近障碍保守膨胀稍微缩小
- 普通膨胀带从“接近必阻塞”降为“规划器可尝试穿过”

运行时日志已明确打印:

```text
fitplane thresholds: max_angle_deg=48.00 max_flatness=0.0200 angle_weight=0.80 occupied_value=100 inflation_value=94 near_obstacle_value=118
```

说明本次放宽不是只改了源码，实际运行已经吃到了新参数。
