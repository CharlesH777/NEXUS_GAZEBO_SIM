## 🔍 根本问题

### TF 树完全断裂

**现状**:
```
NEXUS 提供：
  world → map (静态)
  odom → base_footprint (动态，来自 Gazebo)
  sensor → base_link (静态)
  
但这些是三个独立的树！没有连接！
```

**LRAE 需要**:
```
map → odom → base_footprint → base_link
      ↑                      ↑
      缺失                   缺失
```

**症状**:
```
[WARN] Could not find a connection between 'map' and 'base_link'
[ERROR] Invalid frame ID "map" passed to canTransform
```

## 💡 为什么这么难？

### NEXUS 的假设
- NEXUS 假设你会运行 SLAM (FAST-LIO2)
- SLAM 会发布 `map -> odom` TF
- 没有 SLAM，TF 树就是断的

### LRAE 的假设
- LRAE 假设有完整的 `map -> base_link` TF
- 这在有 SLAM 的系统中自动存在
- 但纯仿真环境需要手动建立

### 集成挑战
- 两个系统设计假设不同
- TF 树依赖需要精确匹配
- Python 环境需要兼容
- 需要深入理解两个系统的架构
